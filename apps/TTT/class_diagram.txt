Tic Tac Toe OOP Class Diagram
=============================

Classes:
--------

1. Player
   - Attributes:
     * symbol: string (e.g., 'X' or 'O')
   - Methods:
     * (constructor)

2. Board
   - Attributes:
     * grid: array of 9 elements (null, 'X', or 'O')
   - Methods:
     * makeMove(index: number, symbol: string): boolean
     * checkWinner(): string | null
     * isFull(): boolean
     * reset(): void

3. Game
   - Attributes:
     * board: Board
     * players: array of Player (2 players)
     * currentPlayerIndex: number (0 or 1)
     * gameOver: boolean
   - Methods:
     * init(): void
     * handleCellClick(event: Event): void
     * updateDisplay(): void
     * resetGame(): void

Relationships:
--------------

- Game --(1)--> Board (composition: Game has one Board)
- Game --(2)--> Player (aggregation: Game has two Players)
- Board --(9)--> Cell (conceptual: Board manages 9 cells, but cells are represented in grid array)

Note: The diagram uses UML-like notation in text form.
- (1) means one instance
- (9) means nine conceptual cells
